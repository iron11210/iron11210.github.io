目标：能写出符合规范的变量名

规则：必须遵守，不遵守报错 (法律层面)

** 规范：** 建议，不遵守不会报错，但不符合业内通识 （道德层面）

<span style="color:red;">js 命名规则 </span>：

  • 不能用关键字
  • 关键字：有特殊含义的字符，JavaScript 内置的一些英语词汇。例如：let、var、if、for 等
  • 只能用下划线、字母、数字、$ 组成，且数字不能开头
  • 字母严格区分大小写，如 Age 和 age 是不同的变量

<span style="color:red;">js 命名规范 </span>：

  • 起名要有意义

  • 遵守小驼峰命名法

    • 第一个单词首字母小写，后面每个单词首字母大写。例：userName

# 5、变量拓展 let 和 var 的区别

let 和 var 区别：

在较旧的 JavaScript，使用关键字 var 来声明变量 ，而不是 let。

var 现在开发中一般不再使用它，只是我们可能再老版程序中看到它。

let 为了解决 var 的一些问题。

var 声明:

  • 可以先使用 在声明 (不合理)
  • var 声明过的变量可以重复声明 (不合理)
  • 比如变量提升、全局变量、没有块级作用域等等

结论：

<span style="color:red;">var 就是个 bug，别迷恋它了，以后声明变量我们统一使用 let </span>

# 2、隐式转换

某些运算符被执行时，系统内部自动将数据类型进行转换，这种转换称为隐式转换。

<span style="color:red;"> 规则：</span>

  • + 号两边只要有一个是字符串，都会把另外一个转成字符串

  • 除了 + 以外的算术运算符 比如 - * / 等都会把数据转成数字类型

<span style="color:red;"> 缺点：</span> 转换类型不明确，靠经验才能总结

<span style="color:red;"> 小技巧：</span>

  • + 号作为正号解析可以转换成数字型

  • 任何数据和字符串相加结果都是字符串

<img src="./images/ 隐式转换.png" style="width: 70%" />

# 3、显式转换

编写程序时过度依靠系统内部的隐式转换是不严禁的，因为隐式转换规律并不清晰，大多是靠经验总结的规律。为了避免因隐式转换带来的问题，通常根逻辑需要对数据进行显示转换。

<span style="color:red;"> 概念：</span> 自己写代码告诉系统该转成什么类型

<span style="color:green;"> 转换为数字型：</span>

  • Number (数据)

    • 转成数字类型
    • 如果字符串内容里有非数字，转换失败时结果为 NaN（Not a Number）即不是一个数字
    • NaN 也是 number 类型的数据，代表非数字

  • parseInt (数据)

    • 只保留整数

  • parseFloat (数据)

    • 可以保留小数

<span style="color:green;"> 转换为字符型：</span>

  • String (数据)
  • 变量.toString (进制)
